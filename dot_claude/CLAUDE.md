# グローバルガイドライン

自律的に判断・実行。確認は最小限に。

## 言語・文体

- 日本語（技術用語は英語のまま）
- 日本語と半角英数字間に半角スペース
- Output Style 指定がある場合はそちらが優先
- 略語：`y`=Yes / `n`=No / `c`=Continue / `r`=Review

## 実行判断

### 即座実行

既存ファイルの編集・修正・テスト実装・依存関係更新など、
影響範囲が局所的で可逆な操作。
重い調査や独立したサブタスクは Sub-Agent に委譲し、メインコンテキストを保護する。

### 確認必須

- 新規ファイル作成・重要ファイル削除
- アーキテクチャ/フォルダ構造の大規模変更
- 新 API・外部ライブラリ導入
- 認証・認可、DB スキーマ変更
- 本番環境・デプロイ設定変更
- **コミット実行（毎回確認。一度許可されても次回は再確認）**

### エラー時

- 実行不可能 → 代替案 3 つ提示
- 部分実行可能 → 可能部分を先行実行、残課題を明示

## コード品質

- 過度な抽象化より明快さを優先
- 早期リターンでネスト削減
- マジックナンバー・ハードコード禁止（定数化/設定分離/環境変数）
- 重複は即座に排除
- 構造変更（リネーム等）と動作変更（機能追加等）を同一コミットに含めない

### 変更管理

変更は以下の 2 種類に明確に分離：

- **構造変更** — コードの配置・整理・フォーマット。動作は一切変更しない
- **動作変更** — 機能の追加・修正・削除。テスト結果が変わる変更

## コンテキスト管理

- 複雑なタスクは「結果だけが重要な純粋タスク」として Sub-Agent で独立実行
- 大規模作業でコンテキスト肥大化 → `/compact` を推奨
